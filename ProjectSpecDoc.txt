Project Manifest & Technical Specification
Project Name: Deepkt
Lead Developer: Adrian Partain
Target Date: Q1-Q2 2026

1. Executive Summary & Purpose
The objective of this project is to build an autonomous, niche-specific music discovery engine that utilizes Digital Signal Processing (DSP) and Vector Embeddings to recommend music based on "Sonic DNA" rather than metadata or popular consensus.

While mainstream platforms like Spotify use collaborative filtering (user behavior), Project Deepkt focuses on Content-Based Filtering. It is designed to solve the "Cold Start" problem for underground sub-genres—specifically Phonk, Dark Trap, and experimental EDM—by identifying tracks with mathematically similar textures, distortion levels, and rhythmic signatures before they have been indexed by major algorithms.

Entrepreneurial Goal
To create a "Vertical AI" tool that provides high-fidelity discovery for power users, curators, and producers in the underground scene. This demonstrates a transition from a consumer-level application to a B2B Music Intelligence tool.

2. Technical Philosophy
The engine operates on the principle of Genre-Agnostic Sonic Analysis. Instead of categorizing by labels, it treats every 30-second audio snippet as a point in a multi-dimensional "Vibe Space."

3. Detailed Execution Roadmap
Phase 1: Data Acquisition (The Crawler) — COMPLETED
Purpose: To build a proprietary dataset from unindexed sources (SoundCloud, YouTube).

Execution: * Utilize yt-dlp to extract raw audio streams.

Snippet Strategy: Capture audio from 00:30 to 01:00 to bypass intros, ads, and tags, focusing on the "drop" or core energy of the track.

Edge Case Handling: Implemented checks for track duration to ensure short files are captured in full.

Phase 2: Feature Extraction (The Analyzer) — IN PROGRESS
Purpose: To translate air pressure (audio) into a mathematical "Signature."

Metrics to Extract:

Tempo (BPM): Establishes the energy baseline.

MFCCs (13 Coefficients): Captures the "timbre" or "vocal texture" of the sound.

Spectral Centroid: Measures "brightness." (Higher = EDM leads; Lower = Dark Trap/Phonk bass).

Zero-Crossing Rate: Measures "noisiness." High values correlate with the specific distortion patterns found in Phonk.

Silence Removal: Use librosa.effects.trim to ensure the analyzer only "hears" the music.

Phase 3: Vector Indexing (The Brain) — UPCOMING
Purpose: To store "Sonic DNA" for lightning-fast comparison.

Tool: ChromaDB (Local Vector Database).

Logic: 1.  Convert the DNA_Vector into an embedding.
2.  Use Cosine Similarity to find the shortest distance between two vectors.
3.  Assign a "Match Percentage" to recommendations to provide "Explainable AI" to the user.

Phase 4: UI & Deployment (The Product) — UPCOMING
Tool: Streamlit.

Features:

A search bar for SoundCloud/YouTube URLs.

A "Vibe Slider" (e.g., "Add 20% more Grit") that adjusts the search vector.

Direct links back to the original platforms (keeping the project legally "Fair Use").

4. Key Competitive Differentiators
Exploration over Safety: Unlike Spotify’s "Daily Mix," which plays songs the user already knows, this engine is tuned to find "Zero-Play" tracks with matching acoustic profiles.

Explainable Recommendations: The UI will detail why a song was recommended (e.g., "Matched on 88% Rhythmic Density").

Low Noise / High Grit: Specifically optimized for sub-genres where distortion is a feature, not a bug.

5. Deployment Hardware & Environment
Environment: Python 3.10+

Core Libraries: yt-dlp, librosa, chromadb, numpy, ffmpeg.

Target Machine: Local dev environment (UO Talapas or personal workstation).

Developer Note for AI: When referencing this project, prioritize "Signal Processing Accuracy" and "Niche Discovery" over broad-market appeal. The goal is to prove technical mastery of complex, unorganized data.
